User Manual

How to use this powerful tool

Map

The aim of the project is displaying geographical data regarging Divvy bikes in the time period that goes from July to December 2013. For that reason it is map centered: evey data is geolocalized on the map that cover the entire surface of the application. An overlay layer of semi-transparent controls is present to let the user iteract with the application. Every control is present only when the user need it. In every moment the user can move the map, zoom in for a more detailed view or zoom out for see more general patterns.

Single map mode

In this mode the user can display all the graphs and compare the data using different charts (that are present at the same time). The user can choose:

  • A day of the year
  • An hour of the day
  • The gender: male, female
  • The user type: customer, subscriber
  • An age range
All the filters are independent and the user can enable/disable them in any order (except for the hour that can be only choosen after the day).

Multi map mode

In this modality the user can compare data for more than one day. The data is displayed directly on the maps, there are no constraint on the number of stations selected or on the day and hour choosen. In this mode every functionality is replicated twice for maintain the separation between the two maps and allow the best user experience possible. The two maps can be also controlled by different person because the controllers behind them are completely separated and can manage more than one workflow at the same time (for the moment only one user per map is supported).

Stations representation

In order to represent Divvy bikes stations there are markers on the location. This markers represent the actual location of every single station, and has a color according to the visualization mode selected by the user. There are three different station visualization modality:

  • Popularity visualization: every station has its own popularity, computed as the number of travels over the period of time and normalized over the maximum (of the stations in city). If the popularity is near 100% the marker will be completely blue, if the popularity is near 0% it will be white. This visualization modality is active by default and doesn't require any use interaction.
  • Inflow visualization: the user has previously activated the inflow using the inflow button on station control. In this modality every station has a marker with colors that goes from white (no flow from current station to the selected one) to red (every bikes that goes in the selected station comes from the current one).
  • Outflow visualization: it is exactly the same as inflow visualization but with the flow of bikes that leave the selected station.

Community areas representation and selection

In order to falicitate the station selection there's the possibility to enable the community areas directly on the map. To enable CA the user has to click on Show CA from selection control.

When CAs are enabled the user can click on one of them to select (multiple selection) all the stations inside it. On mouseover the community area will change color in order to provide the best user experience possible.

Bikes patter representation

This visualization modality is active when the user select the day using calendar control and the desired hour using day control. This type of representation is focused on trips. Every trip that started before the selected hour and is not yet terminated is displayed on the map. The actual position of the bike is obtained interpolating the start station and the end station coordinates and it is displayed as a red circle.

If the user zoom in he/she can actually distinguish every bike and every path. While the user zoom out it is possible to see general patterns. This type of visualization is very useful in combination with moment of day chart and bikes out per hour of the day chart (all visible in the same moment on the screen). User can turn on and off the stations that he/she want to include in the pattern visualization selecting them in station control (compare all modality) or using the community are selection mode.

Controls

Controls are the graphical elements that let the user interact. They are designed in a minimalistic style in order to provide only the useful functions in a intuitive and straightforward way. There are two different type of controllers:

  • Active controllers: that enable the user to request information (that the application will display in different way). They can be hidden when the user don't need them.
  • Enable controllers: this kind of controllers enable/disable the visualization of other controllers and for that reason are always visible.

    Day control

    This controller displays details regarding the selected day (previously selected by the user with the calendar control). The details are:

    • Selected date
    • Weather of the day
    • Sunrise and sunset
    • Temprerature (in °F)
    Additionally it contains also a slider that enables the user to select an hour of the day. If the user move the slider automatically the map will display trips that are active at that time and also the percentage of completion for every trip. The trip informations are displayed directly on the map, every bike is a red circle, and its path is marked with a black line. The portion of path already completed is in red. The path is calculated interpolating the initial station and the final station coordinates and estimate the position of the bike at that time. When that user select the hour automatically the weather and the temperature will be adjusted to the nearest measure.

    This controller contains also a button that hides it from the map. When the day controller is hidden also the bikes visualization will be hidden.

    Charts

    Charts display information about the selected stations. Since there are two different comparison modalities they will adapt to the selected one:

    • Compare all: the user can select unbounded number of stations and it can compare all of them in the same time. Every trip is intended from and to the selected stations. See the explanation of every graph (that you can find below) for details on how the information is displayed. Compare two: the user can select up to two stations and all graphs will display only trips between this two stations. This modality enable the possibility for the user to pick every couple of stations and see fluw from one to the other.

    There are two category of charts:

    • Static charts: displays information about general statistical information about bikes and trips. This informations are not dependent from the date, and for that reason are called static. The user can also apply filters on this graphs using the filters controller.
    • Dynamic charts: displays information about the trips in different formats simultaneusly. This graphs depends on the selected date (if there's one) and on the active filetrs.

    Bikes out per day of the week

    A bar graph representation gives the trend of bike usage spread across the seven days of the week for the overall data. This view helps the user to look for patterns in usage over the week. For instance, some stations record a very low bike usage during the weekend. This chart is one of the dynamic charts. When more than one station is selected, this chart shows the trend of the cumulative data of all the selected stations. Data in this case is more in the form of bins rather than being continuous. We decided to use bars to display this data as bars are more suited to represent bins of data(in this case, data for each day)

    Day of the Week

    Bikes out per hour of the day

    This chart gives a usage trend of bikes for selected stations as a spread over the 24 hours of the day for the overall data. It is a multi-line graph where each line represents the usage of bikes for one of the selected stations. In this form, the data is more continuous as we have a clear idea of usage from one hour of the day to the next. Thus we have used line graph to show this continuity. When multiple stations are selected, lines representing different stations are given different colors to help the user easily distinguish between different stations. However, as the number of stations increase(and colors increase in turn!) a few issues arise. Too many colors can adversely affect the ease with which the users differentiate between stations. Even if the user somehow gets beyond the issue of dealing with many colors and looking them up from the legend, it becomes impossible to show legends for these colors in a limited space. For this reason, we remove color information from these graphs(even other graphs where colors represent stations) when the number of stations go beyond a reasonable limit. However, in order to allow the user to still distinguish between data from different stations, we have added an element of interactivity to these graphs. When the user clicks on a line in the chart, the corresponding station is highlighted on the map.

    Imbalance chart

    This chart shows the relative difference in the inflow and outflow information for the selected stations in a bi-directional bar graph. Each bar represents the difference between inflow and outflow for one station. If the inflow is more than the outflow, the bar extends upwards to an extent proportional to the difference and it extends downwards if the outflow is more. The data for the imbalance is sorted and the data of stations is displayed in descending order. If many stations are selected, then ten stations that have the largest imbalances are displayed.

    Demographic pie charts

    These charts show general information about the rider demographic. One of the charts shows the break up of riders on the basis of gender. The second one shows the break up of riders on the basis of age, where the riders are grouped into different age bins. The third chart shows the ratio of subscribers versus pay-and-use customers. These graphs show cumulative data over all the selected stations.
